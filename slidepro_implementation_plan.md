# [구현 계획서] 프리젠테이션 이노베이터 앱 - SlidePro (가칭)

본 문서는 Windows 기반의 직장인, 강사, 그리고 초중고 학생들을 위한 혁신적인 화면 판서 및 발표 보조 도구 **'SlidePro'**의 빠른 MVP 개발을 위한 기술 구현 계획서입니다. 기존 시장의 한계를 극복하는 두 가지 킬러 기능(무중단 스마트 줌, 타겟별 듀얼 모드)을 중심으로 설계되었습니다.

---

## 1. 프로젝트 개요
* **프로젝트명:** SlidePro (슬라이드프로)
* **플랫폼:** Windows 10 / 11 (데스크톱 네이티브 애플리케이션)
* **핵심 가치:** 발표자의 '생각의 속도'와 청중의 '시선의 속도'를 일치시키는 발표 액셀러레이터
* **개발 목표:** 4주 이내 가볍고 안정적인 핵심 기능(MVP) 구현 및 시장 검증

---

## 2. 타겟 맞춤형 핵심 기능 (듀얼 아키텍처)

### 2.1. [Biz 모드] 직장인 및 전문 강사용 (프로페셔널 & 미니멀)
1.  **무중단 스마트 줌 (No-Break Smart Zoom)**
    * **문제 상황:** PPT 슬라이드 쇼(F5) 도중 특정 표나 엑셀 데이터를 확대하려면 `ESC`를 눌러 편집 모드로 나가거나 별도 돋보기 툴을 켜야 함.
    * **해외 방식:** 단축키(`Win` + `+`) 입력 시, 현재 마우스 커서가 위치한 지점을 중심으로 화면을 부드럽게 확대(GPU 가속 크롭/스케일). PPT 쇼 모드를 이탈하지 않고 확대된 화면 위에 즉시 추가 판서 가능.
2.  **의도 인식 스마트 스냅 (Smart Snapping)**
    * 마우스로 삐뚤빼뚤하게 선을 긋거나 사각형을 그려도, 알고리즘이 이를 감지하여 정밀한 직선 형광펜이나 반투명 하이라이트 박스로 0.1초 만에 자동 보정.
3.  **맥락 인식형 스마트 소멸 (Contextual Auto-Fade)**
    * 판서 후 마우스 커서가 해당 영역을 벗어나거나 다음 장표로 이동하면, 이전에 그린 낙서가 2~3초 뒤 자연스럽게 페이드아웃되어 발표 흐름을 방해하지 않음.

### 2.2. [Fun 모드] 초중고 학생 및 어린이 교육용 (인터랙티브 & 게이미피케이션)
1.  **칭찬 리워드 이펙트 (Physical Reward Effect)**
    * 학생이 정답을 맞히거나 중요 포인트를 강조할 때, 단축키나 툴바 클릭 한 번으로 화면 전체에 **화려한 종이가루(Confetti) 폭죽**이 터지거나 폭죽 애니메이션 연출.
2.  **디지털 스탬프 툴**
    * 화면 원하는 곳에 "참 잘했어요!", "집중!", "정답!" 등의 움직이는 애니메이션 스탬프(GIF/Lottie 연동)를 쾅 찍어 시각적 재미 극대화.
3.  **게이밍 사운드 효과**
    * 시각 효과와 연동된 경쾌한 효과음(띵동, 팡파르, 띠로리 등)을 제공하여 시청각적 몰입도 유도 (옵션으로 On/Off 가능).

## 3. 기술 스택 및 시스템 아키텍처

저사양 사무용 노트북에서도 Zoom/Teams 비디오 송출과 PPT 슬라이드 쇼를 동시에 돌려야 하므로 **초경량·고성능 아키텍처**를 채택합니다.

* **애플리케이션 프레임워크:** Tauri (Rust + 웹 기술)
    * *이유:* Electron(용량 > 100MB, 메모리 대량 소비) 대비 용량이 수 MB 수준으로 극도로 가볍고 CPU/메모리 점유율이 낮아 Windows 비즈니스 환경에 최적.
* **프론트엔드 UI:** React + TypeScript + Tailwind CSS
    * Tauri 가상 창 위에서 빠르게 렌더링되는 투명 HTML5 Canvas 활용.
* **백엔드 코어 (Rust):**
    * Windows Win32 API 제어 (화면 캡처, 가상 윈도우 스타일 투명화 처리, 마우스 이벤트 관통/포착 제어).
    * 다중 모니터 및 빔 프로젝터 연결 시 DPI 스케일링 자동 보정.

### 시스템 윈도우 레이어 구조
```
+-------------------------------------------------------+
|  [Layer 3] Tauri 프론트엔드 투명 캔버스 (판서/이펙트) |  <- 항상 최상단 (Always on Top)
+-------------------------------------------------------+
|  [Layer 2] 마우스 커서 하이라이트 / 스마트 줌 뷰어   |
+-------------------------------------------------------+
|  [Layer 1] 파워포인트 슬라이드 쇼 (PPT F5 Mode)        |  <- 대상 화면
+-------------------------------------------------------+
```

## 4. UI/UX 디자인 설계 방침

1.  **미니 플로팅 툴바 (자석식 컴팩트 UX)**
    * 화면 좌/우/하단 가장자리에 마우스를 가져가면 스르륵 나타나는 반투명 도크(Dock) 형태.
    * Biz 모드와 Fun 모드를 원터치로 스위칭할 수 있는 토글 버튼 항시 노출.
2.  **단축키 HUD (Heads-Up Display)**
    * 직장인들이 단축키를 외우지 않아도 되도록, 특정 기능 제어 키를 누르고 있을 때 화면 중앙 하단에 현재 활성화된 가이드라인 유저 인터페이스를 세련되게 표시.
3.  **마우스 관통 모드 (Click-Through)**
    * 판서 모드가 꺼져 있을 때는 투명 캔버스가 마우스 클릭을 방해하지 않고 아래에 있는 PPT나 웹 브라우저를 그대로 클릭 및 제어할 수 있도록 Rust 레벨에서 `WS_EX_TRANSPARENT` 윈도우 속성 동적 제어.

## 5. 4주 MVP 개발 마일스톤

| 주차 | 주요 개발 내용 | 완료 기준 |
| :--- | :--- | :--- |
| **1주차**<br>(코어 셋업) | - Tauri + Rust + React 기본 개발 환경 구성<br>- Windows 최상단 투명 윈도우 레이어 구현<br>- HTML5 Canvas 기본 선 그리기 및 지우기 매커니즘 검증 | PPT 화면 위에서 마우스 드래그로 끊김 없이 선이 그려지는 상태 |
| **2주차**<br>(Biz 기능) | - 무중단 스마트 줌(화면 부분 캡처 및 GPU 가속 확대 프레임) 구현<br>- 스마트 스냅(보정 알고리즘) 및 자동 소멸 타이머 구현<br>- 다중 모니터 좌표계 보정 | PPT 슬라이드 쇼 모드를 유지한 채 특정 영역 확대 및 자동 소멸 판서 완료 |
| **3주차**<br>(Fun 기능) | - Canvas 기반 Confetti(종이가루) 폭죽 이펙트 라이브러리 연동<br>- 참 잘했어요 스탬프 및 애니메이션 팝업 구현<br>- 비즈/펀 모드 스위칭 플로팅 툴바 UI 완성 | 툴바에서 모드 전환 후 클릭 시 종이가루 이펙트와 효과음이 정상 출력되는 상태 |
| **4주차**<br>(패키징/테스트)| - Windows 스토어 배포용 패키징 (.msi / .exe)<br>- 사양별 업무용 PC 및 대면 빔 프로젝터 호환성 테스트<br>- 배포용 랜딩 페이지 구축 및 MVP 출시 | 설치 파일 실행 및 실무 환경 테스트 최종 패스 |

## 6. 리스크 관리 및 핵심 점검 사항

* **보안 프로그램 오탐지 리스크:** 직장인 PC의 경우 강력한 사내 보안 프로그램(DRM, 백신)이 최상단 오버레이 앱을 악성코드로 오인할 수 있음. 개발 초기 단계부터 유효한 코드 사인(Code Signing) 인증서 적용 검토.
* **PPT 쇼 모드 포커스 유실:** 간혹 최상단 투명 창이 포커스를 가져가면서 PPT 슬라이드가 단축키(방향키, 스페이스바)로 넘어가지 않는 현상 발생 가능. Tauri 백엔드에서 키 이벤트를 Hooking하여 하위 PPT 윈도우로 이벤트를 바이패스(Bypass) 해주는 로직 확보 필요.
